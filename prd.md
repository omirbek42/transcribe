# PRD: Агент транскрибации аудио/видео (RU/KZ/EN)

## 1. Обзор

**Продукт:** внутренний агент для транскрибации аудио- и видеозаписей
(совещания, интервью, публичные выступления) на русском, казахском и
английском языках, включая смешанную казахско-русскую речь в пределах
одного предложения, с автоматическим приведением текста к корпоративному
деловому стилю, принятому в АО «Самрук-Қазына» и группе компаний фонда.

**Формат реализации:** MVP, open source компоненты, минимальная
инфраструктура, без выделенного контура/сертификации на первой итерации.

## 2. Проблема

Ручная транскрибация совещаний и записей отнимает много времени, а готовые
облачные сервисы либо плохо распознают казахский язык и переключение
кодов (RU↔KZ), либо не знают корпоративной терминологии и названий
портфельных компаний фонда, из-за чего итоговый текст требует ручной
доработки.

## 3. Цели

1. Автоматически превращать аудио/видео в текстовый транскрипт с точностью,
   достаточной для использования без сплошной вычитки.
2. Корректно обрабатывать смешанную казахско-русскую речь без потери
   и без "перевода" одного языка в другой.
3. Автоматически приводить текст к деловому стилю и корпоративной
   терминологии (названия компаний, ФИО, аббревиатуры).
4. Выдавать результат в готовых для использования форматах (протокол,
   чистый текст, субтитры).
5. Запуск прототипа — в течение нескольких дней, без закупки/согласования
   тяжёлой инфраструктуры.

## 4. Вне рамок MVP (Out of scope)

- Работа в закрытом контуре РК / запрет выгрузки данных за рубеж
  (зафиксировано как возможное требование для будущей итерации, не в MVP).
- Real-time/потоковая транскрибация (только batch-обработка готовых файлов).
- Автоматический перевод между языками.
- Полноценная аутентификация пользователей и ролевая модель доступа.
- Развёртывание в production-контуре, мониторинг, SLA.

## 5. Пользователи и сценарии использования

- Сотрудник фонда загружает запись совещания → получает протокол в .docx
  с разбивкой по спикерам и таймкодами.
- Сотрудник загружает интервью/выступление → получает чистый текст для
  публикации или субтитры (.srt) для видео.

## 6. Функциональные требования

| # | Требование | Приоритет |
|---|---|---|
| F1 | Загрузка файла аудио (mp3, wav, m4a) или видео (mp4) через веб-интерфейс | Must |
| F2 | Извлечение аудиодорожки из видео | Must |
| F3 | Распознавание речи на ru/kk/en с автоопределением языка по фразам | Must |
| F4 | Корректная обработка смешанной казахско-русской речи внутри предложения | Must |
| F5 | Диаризация спикеров (разметка "кто говорит") | Should |
| F6 | Коррекция транскрипта вторым проходом через LLM: исправление ошибок распознавания, деловой стиль, глоссарий терминов | Must |
| F7 | Подключаемый глоссарий терминов (названия компаний, аббревиатуры, ФИО), редактируемый без изменения кода | Must |
| F8 | Экспорт результата: .docx (протокол), .txt (чистый текст), .srt (субтитры с исходными таймкодами) | Must |
| F9 | Пометка нечитаемых/неразборчивых фрагментов вместо додумывания текста | Must |
| F10 | Работа на CPU при отсутствии GPU (с понижением модели для скорости) | Should |

## 7. Нефункциональные требования

- Простота развёртывания: venv + `requirements.txt`, без Docker/оркестрации
  на этапе MVP.
- Время обработки: не является приоритетом на MVP-этапе (точность важнее
  скорости), но пайплайн не должен быть заведомо неоптимальным (batch,
  не файл-по-файлу вручную).
- Данные не должны отправляться в LLM/ASR провайдеров, не указанных в
  разделе 9, без явного решения владельца продукта.

## 8. Архитектура / пайплайн

```
[Аудио/видео файл]
   → (если видео) извлечение аудио через ffmpeg
   → ASR (faster-whisper, large-v3) → сырой транскрипт с таймкодами
   → (опционально) диаризация спикеров (pyannote.audio)
   → сборка сырого транскрипта с метками спикеров и таймкодами
   → LLM-коррекция (Claude API, system prompt + glossary.json) → чистый текст
   → экспорт: .docx / .txt / .srt
```

## 9. Технологический стек

- Python 3.11
- ASR: `faster-whisper` (CTranslate2), модель `large-v3` на GPU / `medium`
  на CPU по автодетекту доступности CUDA
- Диаризация: `pyannote.audio`, pipeline `speaker-diarization-3.1`
  (требует HuggingFace token)
- Коррекция стиля: Anthropic API, модель `claude-sonnet-4-6`
- Интерфейс: `streamlit`
- Аудио-утилиты: `ffmpeg`

## 10. Глоссарий и корпоративный стиль

Хранится в `glossary.json`, структура:

```json
{
  "companies": [],
  "abbreviations": {},
  "people": []
}
```

Заполняется владельцем продукта отдельно от кода. Подставляется динамически
в system prompt LLM-коррекции.

## 11. System prompt для шага коррекции

```
Ты — редактор-корректор транскриптов для корпоративного документооборота
АО «Самрук-Қазына» и группы компаний фонда. Тебе передают сырой автоматический
транскрипт совещания/интервью на русском, казахском или смешанном
русско-казахском языке (переключение кодов внутри одного предложения — обычная
практика, не ошибка).

Твоя задача:
1. Исправь ошибки распознавания речи, опираясь на контекст (омофоны,
   неправильно услышанные термины, слитые/разорванные слова).
2. Приведи текст к официально-деловому стилю: убери слова-паразиты, повторы,
   незаконченные фразы — но НЕ меняй смысл и не добавляй информацию, которой
   не было в оригинале.
3. Корректно оформляй названия и термины по глоссарию ниже, расширяй по
   аналогии, если встречаешь похожие названия ДЗО фонда.
4. Названия госорганов РК пиши полностью при первом упоминании, далее —
   принятой аббревиатурой.
5. Сохраняй структуру: если это протокол совещания — оформи по схеме
   «Спикер — реплика», сохрани таймкоды в квадратных скобках в начале
   каждого блока.
6. Если фрагмент нечитаем/неразборчив — помечай [неразборчиво], не выдумывай
   содержание.
7. Не переводи казахские фрагменты на русский и наоборот — сохраняй язык
   оригинала реплики.
8. Выведи только исправленный текст, без комментариев и пояснений.

Глоссарий терминов (JSON, следуй строго):
{glossary_json}

Сырой транскрипт:
{raw_transcript}
```

## 12. Структура репозитория (deliverable)

- `pipeline.py` — извлечение аудио → ASR → диаризация → LLM-коррекция → экспорт
- `glossary.json` — редактируемый словарь терминов
- `app.py` — Streamlit-интерфейс поверх pipeline.py
- `requirements.txt`
- `README.md` — установка (venv, ffmpeg, опционально CUDA), запуск
  (`streamlit run app.py`), где вписать `ANTHROPIC_API_KEY` и `HF_TOKEN`

## 13. Критерии готовности MVP (Definition of Done)

- Загрузка файла → получение .docx/.txt/.srt работает end-to-end на тестовой
  записи с русско-казахским переключением языков.
- Термины из `glossary.json` корректно подставляются в итоговый текст.
- Пайплайн запускается локально без GPU (на пониженной модели).
- README позволяет развернуть проект человеку, не участвовавшему в разработке.
